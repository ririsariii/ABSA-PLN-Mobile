Panduan Sistem ABSA PLN Mobile

Aspect-Based Sentiment Analysis untuk Ulasan Aplikasi PLN Mobile

Tujuan Sistem

Sistem ABSA (Aspect-Based Sentiment Analysis) PLN Mobile dirancang untuk menganalisis ulasan pengguna aplikasi PLN Mobile secara otomatis berdasarkan aspek-aspek spesifik. Sistem ini membantu:

  • Mengidentifikasi aspek-aspek penting dalam ulasan pengguna
  • Mendeteksi sentimen (positif/negatif) untuk setiap aspek
  • Memberikan insight yang lebih detail untuk perbaikan aplikasi
  • Menganalisis tren sentimen dari waktu ke waktu
Model yang Digunakan: Kombinasi Topic Modeling dan Machine Learning Classifier

Fitur-Fitur Sistem

1. Analisis Teks Tunggal

Fitur ini memungkinkan Anda menganalisis satu ulasan secara langsung dengan memasukkan teks ulasan ke dalam kotak input.

  • Input: Teks ulasan dalam Bahasa Indonesia
  • Output: Tabel aspek dan sentimen yang terdeteksi
  • Cocok untuk: Analisis cepat ulasan individual
2. Analisis File CSV

Fitur ini memungkinkan Anda menganalisis banyak ulasan sekaligus dengan mengunggah file CSV.

  • Input: File CSV dengan kolom id, ulasan, dan tanggal
  • Output: Tabel hasil, visualisasi grafik, dan file CSV hasil analisis
  • Cocok untuk: Analisis batch dan monitoring tren

Alur Analisis Sistem

1 Input Ulasan

Pengguna memasukkan teks ulasan atau mengunggah file CSV

2 Preprocessing

Teks dibersihkan dan pre-processing (text cleaning, case folding, tokenization, normalization, stopword removal, stemming)

3 Topic Modeling

Sistem mengidentifikasi aspek-aspek yang dibahas dalam ulasan

4 Sentiment Classification

Untuk setiap aspek yang terdeteksi, sistem mengklasifikasikan sentimen (positif/negatif)

5 Output Hasil

Hasil ditampilkan dalam bentuk tabel dan visualisasi grafik

Format Input

Format Teks

Untuk analisis teks tunggal, masukkan ulasan dalam Bahasa Indonesia. Contoh:

"Aplikasi PLN Mobile sangat membantu untuk mengecek tagihan listrik. Pelayanan customer service juga cepat dan ramah."
Format CSV

Untuk analisis batch, file CSV harus memiliki struktur sebagai berikut:

  • Kolom wajib: id, ulasan, tanggal
  • Format tanggal: YYYY-MM-DD (contoh: 2026-01-15)
  • Encoding: UTF-8
  • Ukuran maksimal: 16MB
Contoh format CSV: 
id,ulasan,tanggal
1,"Aplikasi bagus, mudah digunakan",2026-01-15
2,"Server sering down, mengecewakan",2026-01-16
3,"Fitur pembayaran sangat membantu",2026-01-17

Interpretasi Hasil Analisis

Tabel Hasil

Tabel hasil menampilkan:

  • Aspek: Kategori yang dibahas (misal: User Experience, Service Quality)
  • Sentimen: Penilaian terhadap aspek tersebut (Positive/Negative)
  • Tanda "â€“": Aspek tidak terdeteksi dalam ulasan tersebut
Visualisasi Grafik

Untuk analisis CSV, sistem menyediakan beberapa visualisasi:

  • Bar Chart: Distribusi jumlah sentimen per aspek
  • Pie Chart: Proporsi sentimen positif vs negatif secara keseluruhan
  • Line Chart: Tren sentimen per aspek berdasarkan tanggal
Tips: Gunakan tren line chart untuk memantau perubahan sentimen pengguna dari waktu ke waktu dan mengidentifikasi kapan terjadi peningkatan atau penurunan kepuasan pengguna.
